## 🛠 Ключевые темы ЛР-5

### 1. Функции как аргументы (Передача стратегий)
В классе `ClientCollection` реализован один гибкий метод `.sort_by()`, который принимает внешнюю функцию-стратегию:
* `sort_by_name` — строит клиентов по алфавиту.
* `sort_by_balance` — строит клиентов по толщине кошелька.
* `by_credits_and_balance` — сложная стратегия (сортирует сначала по количеству долгов, а при равенстве — по балансу).

### 2. Функции высшего порядка (`Filter`, `Map`, `Reduce`)
Вся магия обработки списков клиентов в нашей коробке (`ClientCollection`) завязана на каноничных функциях:
* **Фильтрация (`filter_by`):** принимает функцию-предикат и отсеивает неугодных (например, оставляет только тех, у кого баланс строго `> 0`).
* **Трансформация (`map_to_list`):** потрошит клиентов и превращает их в плоский список нужных полей (например, выдергивает только текстовые имена).
* **Схлопывание (`reduce_to_value`):** перебирает коллекцию и сжимает её в одно итоговое число (подсчитывает общую сумму бабла во всем банке).

### 3. Замыкания (Фабрика функций)
В модуле `strategies.py` реализована функция-завод `make_debt_filter(min_debt)`. Мы даем ей планку долга, а она возвращает нам настроенного «агента-ищейку» — внутреннюю функцию, которая запомнила это число и теперь ищет злостных неплательщиков по кредиткам.

### 4. Паттерн «Стратегия» через `Callable`-объект
Класс `AnnualTaxStrategy` (налоговая инспекция) реализует магический метод `__call__`. Это позволяет создавать объекты, которые ведут себя как обычные функции. Мы передаем этот объект прямо в метод `.apply()`, чтобы массово обобрать клиентов на 10% от их баланса.
