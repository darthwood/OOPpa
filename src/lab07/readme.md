# ЛР-7 — Интерактивное консольное приложение «Золотой Насос»

## 1. Цель работы
Объединить все архитектурные знания, полученные в ходе ЛР1–ЛР6, в единое полноценное работающее CLI-приложение, разделенное на слои, с поддержкой автосохранения данных, кастомных исключений и валидации ввода.

---

## 2. Структура проекта
Проект строго разделен на три независимых слоя согласно архитектурным канонам:
* **`main.py`** — Главная точка входа. Запускает приложение, связывает слои, отвечает за триггеры автозагрузки и автосохранения.
* **`cli.py`** — Слой представления (Интерфейс). Содержит только операторы ввода/вывода (`print`, `input`), рисует таблицы и интерактивное меню. Не содержит никакой бизнес-логики.
* **`app.py`** — Слой бизнес-логики. Наш `Generic`-контейнер, управляющий базой данных и выполняющий операции сортировки, фильтрации и поиска.
* **`models.py`** — Описание классов нашей предметной области (`BankAccount`, `Client`).
* **`storage.py`** — Модуль для сохранения коллекции в текстовый формат JSON и её последующего воскрешения при запуске программы.
* **`exceptions.py`** — Файл с нашими собственными уникальными исключениями для обработки логических ошибок банка.

---

## 3. Описание CLI
* **Реализованные пункты меню:** Наше интерактивное меню содержит 7 пунктов (от 0 до 6), позволяющих просматривать красивую форматированную таблицу данных, добавлять уникальных клиентов, удалять записи с подтверждением, искать по ID счета, фильтровать по балансу и менять стратегии сортировки на лету.
* **Обработка некорректного ввода:** Любые попытки пользователя ввести буквы вместо чисел в пунктах меню или полях баланса перехватываются блоками `try-except ValueError`, программа выдает предупреждение и аккуратно возвращает пользователя в меню, предотвращая падение.
* **Сохранение и загрузка данных:** При старте скрипта `main.py` лезет в файл `bank_database.json`, автоматически парсит его структуру и наполняет коллекцию. При выборе пункта меню `0` (Выход) приложение собирает текущий список объектов, переводит в JSON-формат и перезаписывает файл.
