# ** Ответы на 5 вопросов**
1. Что является сущностью?
В представленном коде реализована система банковского обслуживания, состоящая из нескольких взаимосвязанных сущностей:

Transaction: Запись о конкретной финансовой операции.

Credit/Deposit: Финансовые продукты с определенными условиями (ставка, сумма).

BankAccount: Ядро системы, управляющее балансом и историей операций.

Client: Сущность верхнего уровня, представляющая пользователя, который владеет счетом и пользуется банковскими продуктами.

2. Какие у них атрибуты?

Transaction: тип операции (t_type), сумма (amount).

Credit: сумма (amount), процентная ставка (rate), статус оплаты (is_paid).

Deposit: сумма (amount), ставка (rate).

BankAccount: название банка (bank_name), ID счета (acc_id), текущий баланс (balance), история транзакций (history), статус заморозки (is_frozen).

Client: имя (name), привязанный счет (account), списки активных кредитов (credits) и депозитов (deposits).

3. Какие есть инварианты?
Инварианты обеспечивают логическую целостность данных:

ID счета: не может быть короче 5 символов.

Параметры кредита: сумма и ставка всегда должны быть строго больше 0.

Имя клиента: не может быть пустой строкой.

Лимит долга: если баланс опускается ниже -50,000$, счет автоматически замораживается.

Доступ к операциям: любые изменения баланса запрещены, если счет находится в состоянии заморозки.

4. Когда два объекта считаются равными?

В данном коде логика сравнения реализована для сущности Client:

Два объекта Client считаются равными (__eq__), если они оба являются экземплярами класса Client и имеют одинаковый ID банковского счета (acc_id). Это позволяет идентифицировать клиента по его уникальному финансовому идентификатору, даже если другие данные различаются.

5. Есть ли у объекта состояние?

Да, объекты обладают выраженным состоянием:

BankAccount: состояние определяется балансом и флагом is_frozen. Замороженный счет меняет поведение метода change_balance, блокируя операции.

Client: состояние зависит от текущего баланса его счета и наличия задолженностей. Например, метод take_loan откажет в кредите, если состояние 
счета клиента отрицательное.

Credit: имеет логический признак is_paid, определяющий, погашен долг или нет
