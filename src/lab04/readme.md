## 2. Ключевые темы ЛР-4

### 1. Интерфейсы как законы банка (Файл `interfaces.py`)
Чтобы наши банковские счета не устраивали анархию, мы ввели два строгих контракта с помощью модуля `abc`:
* **`Printable` (Контракт на публичность):** обязывает класс реализовать метод `display()`. Теперь каждый счет должен уметь внятно и красиво отчитываться о своем состоянии в консоль.
* **`Comparable` (Контракт на иерархию):** требует метод `compare_with(other)`. Он нужен, чтобы банк мог безболезненно попарно сравнивать балансы счетов, не ломая логику.

### 2. Множественная реализация интерфейсов в моделях (Файл `models.py`)
Наши финансовые инструменты — `SavingsAccount` (Сберегательный) и `CreditCard` (Кредитка) — теперь официально подписали эти контракты и реализуют их:
* **Полиморфизм в `display()`:** Каждый класс выводит информацию по-своему, учитывая свои уникальные фичи (проценты или кредитный лимит).
* **Полиморфизм в `compare_with()`:** Сравнение идет по принципу «у кого больше денег на счету, тот и босс». Метод возвращает текстовый вердикт сравнения текущего счета с любым другим.

### 3. Универсальные функции и интеграция с коллекцией
Наш код в `demo.py` написан по канонам чистой архитектуры. Функции заточены под контракты:
* `print_any_objects(items: list[Printable])` — принимает только то, что можно напечатать.
* `compare_all_pairs(items: list[Comparable])` — принимает только то, что умеет сравниваться.
* Фильтрация коллекции происходит динамически с помощью `isinstance(obj, Printable)`.
